SEO investigation report –[5/13/2026]
This report presents the results of the website's SEO optimization. This report provides the findings from the SEO optimization of the website.

Introduction

AIESEC Canada Hub and AIESEC.ca are two of the main sites for AIESEC Canada. The sites have been optimized after cloning and temporarily moved to Render, this is described in this report. The optimization was conducted with the aim of boosting the site's ranking in Google and making it easy to be found by the search engine's crawlers.

Technical SEO
The initial step taken to optimize the site was to make use of a robots.txt file. A plain text file located at /robots.txt that communicates to search engine crawlers which sections of the website they can crawl. All pages were made accessible for all crawlers and the file also references the sitemap, allowing Google to crawl it automatically.

An XML sitemap was also produced and served at /sitemap.xml. A sitemap is a file that provides a list of all the most important urls of the site which is given directly to Google, instead of going by the links provided on the website. This way, no pages will be missed while indexing. The robots.txt and sitemap were both made into routes and no file would need to be stored on disk.

All pages have been provided with canonical tags in the base template. A canonical tag is a particular HTML element that specifies the appropriate URL for a page. This was especially crucial for this project, as the original AIESEC Canada site is in existence with the same content — the canonical tag alerts Google to the fact that this is a separate site, thereby avoiding duplicate content penalties.

All static files (images, CSS, JavaScript) have Cache-Control headers. These headers tell the browser to keep a locally stored copy of all static files for 7 days – so visitors coming back to the site don't waste time re-downloading the same files every time. This has a direct impact on page speed and is a proven Google ranking factor.


On page SEO
Each page had unique title tags and meta descriptions. Before this, all pages were sending "nil" data to the Go templates, causing titles to be empty or the same everywhere in the site. In Go, a struct called `PageData` was added that allows for passing a different title and meta description per page. The title tag is the blue-link appearing in Google's search results and the meta description is the text that appears below — both of these elements have a direct impact on CTR.

Base template was enhanced with Open Graph tags. They are HTML metatags which define the title, description and image preview for a page when shared on social media such as LinkedIn or Twitter. This increases the visibility of the site, not just in Google's search engine but on the internet as a whole.

All pages have been audited and corrected to have a hierarchy of the header. All pages had one `<h1>` tag for the main theme of the page with the use of `<h2>` and `<h3>` tags for sub-sections under this. Pages which were jumping from `<h1>` to `<h3>` and those that did not have an `<h1>` at all were fixed. Having the proper header hierarchy helps Google understand the structure and relevance of each page's content.


Image Optimization
All images have been added with Alt text on all pages. When you add an image to a page, you can add a short text description to the image tag, which is referred to as Alt Text and helps Google and screen readers understand what the image is. If it doesn't, then Google doesn't have any context for what the image is about. Each alternative text was crafted to be descriptive and specific to the image and not generic placeholders.

A Python script using the Pillow library was used for image compression. In the static folder there are more than 500 pictures, so compression was not possible manually and was written a script to compress all images automatically. All the images were converted to WebP format, 80% quality. WebP is a new image format created by Google that results in much smaller file sizes than JPG and PNG without compromising on the image quality. The outcome was a significant reduction in file sizes, with an average of 60–80% across the entire dataset, which directly benefited page load time for the entire site.

Limitations
This was a cloned site so there were restrictions on the level of optimisation possible. Unfortunately, the content on all pages was not able to be drastically changed and keyword research, content gap analysis, and content depth improvements — all of which are major elements in SEO — were not part of this project. As such, there was of course no backlink strategy to be carried out because the links would need original content to be linked to which would be desired by other sites. Furthermore, some pages remained to be migrated from individual HTML pages to the base template as of this writing and the base template improvements (including meta tag and canonical tags) were not yet present on all pages.
